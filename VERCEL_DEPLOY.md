# Деплой на Vercel — Инструкция

## Шаг 1: Подготовка Git репозитория

```bash
git add .
git commit -m "Add Vercel deployment config"
git push origin main  # или master, зависит от вашей ветки
```

## Шаг 2: Вход на Vercel и импорт проекта

1. Зайдите на **https://vercel.com/**
2. Нажмите **"New Project"** (если не залогированы, сначала логин/регистрация через GitHub)
3. Выберите **"Import Git Repository"**
4. Найдите и выберите ваш репозиторий `Wiru-Combat-Academy-WEB`

## Шаг 3: Настройка переменных окружения

На экране **"Configure Project"** (перед деплоем) добавьте переменные:

### Обязательные:
- **FLASK_APP** = `app.py`
- **SECRET_KEY** = (сгенерируйте длинный случайный ключ)
  - Пример: `your-very-long-random-secret-key-here-min-32-chars`

### Опционально (для продакшена):
- **MAIL_FROM** = `"Wiru Combat Academy <noreply@example.com>"`
- **MAIL_TO** = `your-email@example.com`
- **MAILGUN_API_KEY** = (ваш Mailgun API ключ, если используете почту)
- **MAILGUN_DOMAIN** = `sandbox...mailgun.org`
- **BABEL_DEFAULT_LOCALE** = `ru`

### Если хотите использовать PostgreSQL вместо SQLite:
- **DATABASE_URL** = `postgresql://user:password@host:5432/dbname`
  - Используйте сервис типа Heroku Postgres, Supabase или Railway

## Шаг 4: Деплой

После настройки переменных нажмите **"Deploy"**. Vercel автоматически:
1. Клонирует ваш репозиторий
2. Установит зависимости из `requirements.txt`
3. Запустит приложение с помощью Gunicorn

Процесс займёт 1-3 минуты. После завершения вы получите URL вида `https://your-project-name.vercel.app`

## Шаг 5: Проверка

После успешного деплоя откройте URL и проверьте:
- [ ] Главная страница загружается
- [ ] Страница /trainers работает (должна быть Каролина)
- [ ] Переводы переключаются (?lang=en, ?lang=et)
- [ ] Админ-панель доступна (/admin, если её открыли)

## Важно: ограничения Vercel

### 📌 **Файловая система не персистентна**
- SQLite БД будет сброшена при каждом новом деплойменте (обновления кода)
- **Решение:** Используйте PostgreSQL (см. DATABASE_URL выше)
- Для разработки SQLite в порядке

### 📌 **Максимум 10 секунд на обработку запроса**
- Долгие операции могут упасть
- Обычные запросы работают нормально

### 📌 **Максимум 512 МБ памяти**
- Вполне хватает для Flask приложения

### 📌 **Статические файлы**
- Vercel автоматически сервирует папку `static/`
- Убедитесь, что `url_for()` правильно генерирует пути

## Рекомендации

1. **Используйте переменные окружения** для всех чувствительных данных (API ключи, SECRET_KEY)
2. **Настройте логирование** через `VERCEL_LOGS` или платформу типа Sentry
3. **Регулярно проверяйте логи** в дdashboard Vercel (Deployments → Logs)
4. **Для БД:** если захотите сохранять данные, используйте PostgreSQL (бесплатные варианты: Supabase, Railway, Render)

## При проблемах

Если деплой падает:
1. Откройте **Deployments → [последний деплой] → Logs**
2. Посмотрите ошибку в логах
3. Добавьте нужные переменные окружения (если ошибка про них)
4. Рекоммитьте и пушьте — Vercel автоматически пересоберёт

## Удаление/пересоздание

Если что-то пошло не так:
1. На странице проекта в Vercel нажмите **Settings → Danger Zone → Delete Project**
2. Пересоздайте проект заново (шаги 2-4)
